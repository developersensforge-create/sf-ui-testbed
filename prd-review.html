Feature: Camera Skills Status: Draft Generated: Pages: P1–P7

Overview

Camera Skills is a smart automation feature that allows SensForge camera users to configure if-then rules based on AI detection events. Users define a trigger (e.g., "Person Detected"), optional conditions (e.g., "Unknown person"), select which device to monitor, choose an action (e.g., "Send Notification"), and set a schedule — all without writing a single line of code.

The feature is designed for non-technical homeowners who want intelligent, contextual automation from their existing SensForge camera hardware.

Key Screens

P1 — My Skills Dashboard

[ P1 Screenshot ]

The main entry point. Users see their active skills in a card list, with toggles to enable/disable each rule. A template builder at the top lets users start a new If/Then rule by tapping the trigger pill. Newly activated skills appear at the top of the list and are stored in localStorage.

  • If/Then template builder with pill-based inputs
  • Active skills list with toggle, tags, and edit link
  • Activation flow returns user here via URL params

P2 — Select Trigger

[ P2 Screenshot ]

Users select from 11 AI detection triggers. Single-select radio list. Once selected, the Next button activates and navigates forward, passing the trigger via URL param.

  • Person Detected, Person Coming Home, Person Near Car or Driveway
  • Motion Detected, Vehicle Detected, Package Delivered, Package Taken
  • Familiar Face Detected, Pet Behavior, Unusual Sound Detected, Fall Detected (Elder)

P3 — Add Conditions

[ P3 Screenshot ]

Optional refinement step. Context-aware suggestions appear based on the selected trigger. Users can type a free-form condition or tap suggestion pills. Condition is passed forward via URL param.

  • Free-form text input for custom conditions
  • Dynamic suggestion pills per trigger (e.g., "Unknown person", "At night")
  • Entirely optional — skip by tapping Done without filling

P4 — Select Device

[ P4 Screenshot ]

Users choose which camera to monitor. Device list shows name, status, and an active/inactive toggle. Living Room Cam is pre-selected as the default. Selection passes forward via URL param.

P5 — Select Action

[ P5 Screenshot ]

Users select the action triggered when the detection event occurs. Send Notification is checked by default. If "Play Customized Voice" is selected, a sub-option appears to record a voice memo (linking to P7).

  • Send Notification (default), Record, Turn on Light
  • Play Customized Voice (with voice memo sub-flow), Trigger Siren, Call Me

P6 — Set Schedule

[ P6 Screenshot ]

Users set when the skill should be active. Time range (Start/End time pickers) and repeat schedule (Everyday, Weekdays, Weekends, Custom). Custom reveals day-of-week toggles. Done — Activate Now returns to P1 and creates the skill card.

P7 — Voice Memo

[ P7 Screenshot ]

Optional recording flow for custom voice announcements. Three states: Recording (animated waveform), Paused, and Playback. Save Memo returns to P5 with device/trigger context intact.

Technical Notes

URL parameters thread selections through all pages — each page reads query params for context and appends its selection when navigating forward.
Comments are stored in localStorage per page using the key pattern comments_{pageId}_{timestamp}. The comment system supports timestamped display and deletion.
Skills are stored in localStorage when Activate Now is pressed on P1. The active skills list renders from both the static HTML defaults and dynamically added localStorage entries.
Progress bar state on the dashboard is stored in localStorage per feature key (feature-state-{key}). States: idle → building → complete → idle. Auto-transitions to complete after 30s for demo purposes.

Open Questions

  • Should skills support multiple actions (e.g., notify + record simultaneously)?
  • How should the app handle device offline states in the selection step?
  • Should the schedule step support recurring time windows (e.g., 8–10 AM and 8–10 PM)?
  • Is there a maximum number of active skills per user?
